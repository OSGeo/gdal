JPEGXR -- JPEG XR File Format

The JPEG XR format is supported for reading, and batch writing, but not update in place. JPEG XR files are represented as one band (greyscale) or three band (RGB) or four band (RGB with Alpha) datasets with Byte valued bands.

The driver is implemented based on the jxrlib - open source (BSD) implementation of JPEG XR format by Microsoft.

Driver Features

  • Compression - lossless by default, controlled with QUALITY option.
  • Image Structure - entire image is decoded; tiling not yet supported.
  • Channel Organization - Grey, RGB, BGR, RGBA, BGRA.
  • Color - RBG images YUV with 8-bit per sample and 4:4:4 full chroma resolution encoding by default.

Creation Options

JPEG XR files are created using the "JPEGXR" driver code. JPEG XR file creation is implemented by the batch (CreateCopy) method.

Encoder properties can be controlled with the following options:

  • QUALITY=n: By default the quality flag is set to 100, but this option can be used to select other values. Values must be in therange from 0 (lossy) to 100 (lossless).

  • OVERLAP=[0,1,2]: Overlap processing level
    • 0 - No overlap processing (default for lossless quality)
    • 1 - One level (default for lossy quality), modifying 4x4 block encoded values based on values of neighboring blocks.
    • 2 - Two level, in addition to the first level, encoded values of 16x16 macro blocks are modified based on the values of neighboring macro blocks.

Security Note

JPEG XR format users be aware that fuzzing tests of image format libraries by author of american fuzzy lop revealed jxrlib suffers of "sheer number of exploitable bug" see afl-generated, minimized image test demo.
